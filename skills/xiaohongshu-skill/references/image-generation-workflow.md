# 图片生成工作流

## 什么时候读

当任务需要 Image2、生图提示词、封面图、内页图、整套图文包或图片降级路线时读取本文件。

## 路线顺序

优先级从高到低，不可跳级（权威定义见 `skill-contract.md`「图片路线合同」）：

1. Image2 / 宿主原生图片生成。
2. 其他图片生成 MCP 或插件（豆包、即梦、MiniMax、Midjourney 等）。
3. 已有真实素材处理。
4. SVG、HTML、本地排版 PNG 或静态结构预览。

**硬约束：当第 1 级或第 2 级任一可用时，禁止使用第 4 级。** 只有第 1 级和第 2 级都不可用、都失败，或用户明确要求可编辑文字时，才允许进入第 4 级，并必须写明 fallback 原因和降级证据。调用第 2 级 MCP 前必须先写 `Image2 优先级证据`（Image2 已调用并失败 / 当前宿主无 Image2 可调用工具 / 用户明确要求不用 Image2）。

SVG、HTML、本地排版只能当结构预览，不能写成交付级图片，不能冒充真实图片。

缺真实素材时不要自动停图。用户要真实感、真人写真、留学生活、交友、美女风格或模拟真实素材，就继续写 Image2 Brief 并调用 Image2/宿主原生图像；可以在最终标注“生成素材/可替换为实拍素材”，但不能只给图片建议。

“图片上文字说明是主体”“文字信息卡”“大字封面”“知识卡”不是跳过 Image2 的理由，也不是直通本地排版的理由。Image2 或其他图片 MCP 任一可用时禁止本地排版；先用 Image2/宿主原生图像做一体化图文封面，只有错字、不可读、排版失败或用户要求可编辑文字时，才使用本地排版/overlay fallback，并写清 fallback 原因。

在 Codex 等宿主中，如果没有一个字面名为 `Image2` 的工具，但有宿主原生图片生成能力，宿主原生图片生成就是本技能的 Image2 主路径。可接受证据包括：`imageGeneration` 项、`image_gen` 调用结果、本轮新生成图片路径、宿主可见图片结果。不要因为工具名不是 Image2 就跳过原生生图，也不要在原生生图可用时先调用其他图片 MCP。

历史输出隔离：旧 `outputs/` 目录、旧线程图片、示例文件、缓存图和以前生成的 Markdown 不是本轮图片生成证据。除非用户明确说“复用这套旧图/旧包”，否则它们只能作为参考，不得写成“已生成图片”或“交付级”。

## 可选视觉工作台层

视觉工作台不在上述 1-4 级路线中。它只能在逐页视觉导演和对应 Image2 Brief 已确定后承载页面槽位、插入已经由当前图片路线生成的资产、做移动端缩略图检查，以及组织批注返修对照。它不能作为第 2 级图片生成 MCP，也不能在 Image2 失败后“救场”。

启用时按 `visual-workspace-contract.md` 执行；具体供应商能力先读对应适配清单。工作台不可用时，直接继续原有 Image2 / 原生图片路线，不改变结果状态。

## 生成前确认

出图前必须确认：

- 策略路线。
- 标题和封面字。
- 视觉路线。
- 图片 Brief。
- 封面候选或批量生产许可。

如果用户明确授权“你定/直接做”，记录原话。否则关键节点必须等待用户确认。

确认状态必须来自流程证据，不得写死：

- `human_confirmed`：用户原话选择，或 `request_user_input` / `AskUserQuestion` 返回。
- `explicit_auto_permission`：用户明确要求直接做，并允许模型按推荐路线执行。
- `blocked_pending_human_decision`：缺关键选择且没有原生确认工具或未返回。

不能把“推荐方案”或模型默认选择写成 `human_confirmed`。

生成类请求的确认不是范围降级许可。用户已经要求图文、引流贴、完整发布包、封面/内页或图片生成时，前置确认的作用是锁定方向和 Brief；一旦确认完成，下一步就是生成封面候选。触发本 skill 后没有纯文案或仅脚本推荐路线；用户要求暂停时，只能停在当前流程状态并等待继续/修改/批量确认。

## 图片需求说明

封面 Brief 包含：

- 画幅：小红书 3:4 竖版。
- 主标题和副标题。
- 主体和场景。
- 构图和视觉重心。
- 镜头、光线、材质。
- 文字层级和可读性要求。
- 成功标准。
- 失败后的 fallback。

内页 Brief 还要包含页码、页标题、该页唯一要点、画面角色、素材需求、主体区、主标题区、辅助文字区、禁止遮挡区和与前后页的连续元素。字段权威定义见 `page-art-direction-system.md`。

## 提示词模板

提示词先按逐页视觉导演表装配，再按这个顺序写：

```text
生成 3:4 竖版小红书图文页。
主题：[主题]
页面角色：[封面/第几页/流程页/清单页/故事页]
读者在本页完成的认知动作：[代入/理解/相信/比较/行动]
本页唯一承诺：[一句可兑现结论]
画面主体：[真实物件或场景]
构图：[版式、主体位置、文字区域、留白、阅读方向]
文字：[主标题/副标题/页内短句]；精确位置：[坐标或上/下/左/右留白区]；不能遮挡：[脸/产品细节/证据]
风格：[真实、克制、手机随拍、教育信息图等]
连续性：[与前后页共用的色盘/光线/组件，以及避免重复的元素]
验收：[缩略图先读主标题，文字不贴边，不遮挡主体，与页面角色一致]
```

短中文标题可以要求 Image2 一体化生成。密集中文、表格或多段文字容易错字时，先要求干净底图和明确文字区；只有中文错字、不可读、排版失败或用户要求可编辑时，才走既有本地 overlay fallback，并写明原因。

## 封面验证

封面生成后先检查：

- 图片文件或宿主可见结果是否存在。
- 主标题在手机缩略图里是否可读。
- 文字是否错字、糊字、贴边。
- 用户是否确认继续做内页。

封面未通过，不进入批量内页。

满意流程顺序：

1. 用宿主原生选择确认策略、标题/封面字和视觉路线。
2. 确认完成后直接生成 1 张本轮新封面候选；不要再追问“这次做到哪一步”。
3. 做手机信息流小图判断；不通过时修改 Brief，而不是连续抽卡。
4. 若启用视觉工作台，把封面插入 3:4 槽位做移动端缩略图检查；槽位或画布 id 不能代替检查结果。
5. 封面通过后立即用宿主原生选择让用户选：通过并批量生成 / 先停查看 / 不通过改 Brief。画布内点击或摆放不算批量许可。
6. 用户确认批量生成后，生成封面 + 6 到 8 张内页图。
7. 最终聊天展示封面预览、正文、标题备选、承接、标签和图片清单；完整 15 段工作台只放归档文件。

用户确认批量生成后，后续动作只能是执行批量出图、汇报具体失败页或请求重试/降级许可；不得再弹“这次做到哪一步”、不得把“文案+脚本”作为推荐，也不得把单张封面当完整交付。

## 批量生产

完整图文包通常包含封面 + 6 到 8 页内页。批量生产必须记录：

- 每页图片路径或可见结果。
- 每页标题。
- 每页素材来源。
- Image2 成功/失败状态。
- 降级原因。
- 最终状态：`交付级`、`partial` 或 `blocked`。

如果启用视觉工作台，还必须形成逐页资产清单，为每页记录 `generation_evidence`、`workspace_placement`、`mobile_thumbnail_check`、`revision_lineage` 和最终 `accepted_asset`。只对失败页发起批注返修；新图放在原图旁边，复查通过后再更新该页 `accepted_asset`，不得覆盖原始证据。

如果只生成一张封面，不能声称整套图文完成。

如果最终聊天只交付提示词、旧图路径、结构预览或单张封面，必须写 `partial` 或 `blocked`，不得写“完整包已完成”。如果本轮没有 `imageGeneration` / `image_gen` / 新生成路径证据，也不得写“已生成图片”。
