# 内容机会与营销决策模型

## 什么时候读

完整小红书图文包触发后，在写标题、正文、封面或图片 Brief 前读取。本文件是创意上游：标题、叙事、封面和每页设计都必须从这里往下推，不能先凭感觉选“真实桌面”“大字封面”或固定文案套路。

## 谁负责研究

流量机会由 Skill 自己研究，用户只需要给出主题、产品/服务、目标人群或手头素材。研究顺序：

1. 从主题生成读者会使用的搜索词、场景词、痛点词、比较词和行动词。
2. 用当前可用的网页/浏览器/平台搜索能力查看官方平台资料、公开可见的同类内容、评论讨论和高信号案例。
3. 记录同类内容反复承诺什么、读者在问什么、哪些角度已经过度重复、有什么可被真实材料支持的空缺。
4. 没有检索能力或看不到真实互动数据时，写 `研究缺口`，只把结论标为 `AI 假设`；不得杜撰“高赞”“爆款”或搜索数据。

高互动案例只可抽象为机制，例如“一个清楚承诺 + 一个可见证据 + 页面节奏”，不能复制它的标题、话术、人物、图或版式。

## 决策板

每次完整包先形成以下字段。字段来源只能是 `用户提供`、`研究证据` 或 `AI 假设`。

| 决策 | 要回答什么 | 会改变什么 |
|---|---|---|
| 目标读者与具体场景 | 谁在什么时候遇到什么事 | 标题的人群语言、场景与主视觉 |
| 要抢的需求 | 这条内容解决哪个搜索/浏览需求 | 标题路线、封面承诺 |
| 需求阶段 | 未意识到 / 已知痛点 / 找方法 / 比较 / 准备行动 / 分享 | 叙事结构、证据量、产品露出 |
| 主目标 | 停留 / 收藏搜索 / 评论讨论 / 信任种草 / 咨询 / 转化 | 页面顺序与最后一页动作 |
| 次目标 | 只允许一个 | 信息取舍 |
| 张力 | 明显痛点、隐性担忧、期望结果、害怕付出的代价 | 标题框架和第 1 页 |
| 产品角色 | 不出现 / 背景工具 / 方法载体 / 证据对象 / 对比对象 / 直接方案 | 显性或隐性表达 |
| 证据强度 | 个人经历 / 过程记录 / 可演示使用 / 可核验结果 / 对比依据；以及不能说什么 | 该不该用真实场景、对比页、限制页 |
| 信任距离 | 个人 IP 近 / 新账号中 / 品牌远 | 人称、真实痕迹与证据密度 |
| 表达强度 | 隐性 / 半显性 / 显性 | 产品出现位置和文字力度 |
| 信息框架 | 场景痛点 / 搜索答案 / 判断标准 / 反差观点 / 身份认同 / 低风险行动 | 标题与封面主句 |
| 视觉证据方式 | 真实记录 / 人物情境 / 使用演示 / 产品细节 / 比较表 / 信息图 / 氛围场景 | 每页画什么 |

## 从决策到成品

必须留下这条可检查链路：

```text
内容机会研究
-> 营销决策
-> 标题承诺
-> 封面第一眼任务
-> 正文叙事路线
-> 逐页信息角色与视觉导演表
-> 产品出现方式
-> 发布后的优先观察信号
```

封面只负责让对的人停住；标题补足“为什么与你有关”与必要搜索词；正文负责解释、故事、方法或比较；内页负责让读者依次完成代入、理解、相信、判断和行动。四者不能重复喊同一句话。

## 常见路线

| 内容目标与阶段 | 标题/封面方向 | 内页主线 | 产品表达 |
|---|---|---|---|
| 停留，未意识到 | 身份或场景痛点 + 新判断 | 场景 -> 看见问题 -> 新理解 -> 小动作 | 隐性 |
| 收藏，找方法 | 具体问题 + 可检索答案 | 结果预览 -> 步骤 -> 易错点 -> 清单 | 半显性 |
| 信任，种草 | 真实困扰或微小变化 | 场景 -> 过程 -> 证据 -> 限制 -> 感受 | 隐性或半显性 |
| 比较，准备行动 | 怎么选 / 谁适合 / 别只看某一点 | 人群分型 -> 标准 -> 对比 -> 适用边界 -> 结论 | 显性可接受 |
| 咨询或转化 | 人群 + 明确场景 + 可兑现交付 | 问题 -> 机制 -> 可验证材料 -> 适用/不适用 -> 下一步 | 显性但不夸大 |

真实感、人物、强反差、避坑、信息卡都不是默认正确答案；只有它们能帮助本条完成上面的页面任务时才使用。

## 通过标准

- 没有“机会研究 + 决策板”，不得产出最终标题、封面或批量内页。
- 每个标题、封面与内页路线都能指出它对应的上游决策。
- 同一主题在“停留、收藏、信任、咨询”四个目标下，标题、叙事和页面设计必须明显不同。
