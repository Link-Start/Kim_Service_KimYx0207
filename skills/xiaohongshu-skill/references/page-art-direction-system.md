# 封面与逐页视觉导演系统

## 目标

把“要做一组图”变成可执行的页面脚本。每页先决定读者看完后应发生什么，再决定画什么、字写什么、字摆哪里、怎样生成图。不要每页都做成一张漂亮桌面图加一句话。

## 统一画布与安全区

- 默认画布：`1080 x 1440`，3:4 竖版；整组保持相同比例。
- 生产安全边距：顶部约 120px、底部约 140px、左右约 72px 不放关键文字或关键证据。这是小屏可读性和裁切容错，不是平台官方硬性尺寸。
- 整组固定主字体层级、2-3 个主色、光线/材质逻辑和页码位置；不要每页换一种审美。
- 图片承担场景、过程、证据和情绪；文字承担结论、判断、步骤和条件。不要重复描述图片里已经一眼可见的内容。

## 封面：先分主体区与文字区

封面只能承诺一个读者收益或冲突。主标题 1-2 行，副标题最多 1 行；文字总面积通常不超过画面约三成。文字必须落在低细节留白区或专门信息底板，不能盖住脸、手、产品关键细节、截图证据。

从下列版式中按素材和目标选择，不把任何一种写死：

| 版式 | 主体与文字 | 适合 | 禁止 |
|---|---|---|---|
| A 左文右物/人 | 标题在左上或左中，主体在右下或右侧 | 人物、产品、使用情境 | 标题压主体脸部/关键动作 |
| B 上文下景 | 上方留白写承诺，下方展示完整场景 | 桌面、门店、旅行、过程记录 | 场景过碎、上方没有真实留白 |
| C 大主体留白 | 单一主体占大部，标题落在低细节边缘 | 单品、情绪、结果物 | 用小贴纸补核心信息 |
| D 对照分屏 | 上下或左右呈现两种状态，标题跨中线但避开证据 | 避坑、前后、选择 | 失真前后对比、两边光线尺度不同 |
| E 信息主导卡 | 主结论最大，辅助图只提供证明 | 清单、流程、工具教程 | 用于必须靠真实过程建立信任的内容 |

封面输出必须写：`版式`、`主体区`、`主标题区`、`副标题区`、`不能遮挡区`、`缩略图第一眼读到什么`、`为什么选这套构图`。

## 文字层级与位置

每页最多一个最大信息层：

1. 页码/章节标签：最小，只做定位。
2. 本页结论：最大，在上方或明确留白区，一眼扫读。
3. 解释/证据短句：中等，只补结论。
4. 注释/条件：最小，紧贴所解释的对象，不承担主信息。

真实场景页少字，让画面承担可信度；方法、比较、清单页可多字，但必须有明显网格与层级。不要把正文段落塞在图片里。

## 页面角色库

每套从下列角色组合 6-8 页；不要机械凑齐。每页只能有一个主任务。

| 页面角色 | 读者看完应发生什么 | 推荐画面 | 文字位置 |
|---|---|---|---|
| 代入页 | “这说的就是我” | 具体生活/工作场景与痛点细节 | 主体反侧的留白区 |
| 问题证据页 | 痛点变得具体可见 | 物件、流程、截图式证据、局部特写 | 顶部结论 + 靠近证据的短注释 |
| 新判断页 | 明白一个反常识或转折 | 单一强画面或低干扰信息图 | 大结论在留白区 |
| 方法拆解页 | 能马上做一个动作 | 俯拍/近景，单一步骤 | 上方步骤名，下方短条件 |
| 过程/证据页 | 相信它不是空话 | 过程、使用演示、真实材料 | 少字，证据旁小注释 |
| 对比/选择页 | 知道如何选、何时不适合 | 同光线同角度 A/B 对比或判断表 | 顶部问题，两侧短标签 |
| 风险边界页 | 不被夸大承诺误导 | 限制条件、反例或警示场景 | 信息底板或清晰留白 |
| 收束行动页 | 愿意收藏、评论或采取低门槛动作 | 安静的成果/物件/总结场景 | 左上或上方清单区 |

连续页面要交替“场景/证据/解释”，避免连着三页同类画面。每页记录与上一页的连接点，以及不得重复的画面元素。

## 逐页视觉导演表

封面和每页都必须写以下字段，随后才可生成 Image2 Brief：

```text
页码与页面角色：
读者在本页完成的认知动作：
本页唯一信息承诺：
画面主体与真实感锚点：
构图版式（A/B/C/D/E 或页面专用构图）与选择理由：
主体区：
主标题区 / 精确文字：
副标题或证据区 / 精确文字：
文字层级与位置：
不能遮挡或不能出现：
与前后页连续的元素：
本页 Image2 结构化提示词：
验收：缩略图可读、文字不压证据、只讲一个要点、与页面角色一致。
```

## Image2 提示词装配

先写导演表，再编译成提示词。短中文标题可要求一体化生成；密集中文、表格或多段文字容易错字时，先生成干净底图并把文字区留出来，只有在现有 `image-generation-workflow.md` 允许的错字/不可读/排版失败或用户要求可编辑情形下才走 overlay fallback，并记录原因。

通用骨架：

```text
生成 1080x1440、3:4 的小红书第 [n] 页。
页面角色：[角色]；读者应得到：[认知动作]；本页只承诺：[一句话]。
画面主体：[物件/人物/动作/证据]，位于[主体区]；真实感锚点：[光线/材质/使用痕迹]。
构图：[版式和阅读方向]；在[标题区]保留[比例]的干净低细节区域；
标题精确为“[短句]”，[字重/颜色/层级]；[副文字]位于[位置]。
与前页保持[色盘/光线/组件]连续，但避免重复[元素]。
不要遮挡[关键证据]；不要水印、logo、价格贴纸、杂乱背景、无法阅读的小字、夸张疗效或误导性对比。
```

## 常用页面提示词骨架

下面是从公开图文轮播、移动端排版和图像提示词资料中抽象出来的结构，不是“必爆模板”。先用营销决策选择页面角色，再选骨架；不得为了套骨架改变真实内容。

### 1. 利益点封面

```text
用途：3:4 小红书封面底图。
读者：[目标人群]；具体痛点：[场景]；封面只承诺：[一个明确收益/冲突]。
画面：真实的[人物/产品/结果物]位于[右下/下半]，正在[具体动作]；
[左上/上半]保留干净、对比足够的留白，供 1-2 行中文大标题。
视觉：[真实随拍/克制生活方式摄影/专业干货卡]；色盘[由题材决定]；光线[自然窗光/柔和棚拍]。
不要文字、logo、水印、价格贴纸、杂乱背景、拼贴式元素。
```

### 2. 痛点代入页

```text
用途：第 1-2 页，让读者确认“这说的就是我”。
画面：一个具体而真实的[使用场景]，人物正在[动作]，清楚可见[痛点证据]；
主体在[下半/右侧]，上方或反侧留 35% 平整区域给一句判断。
视觉像真实生活记录，不摆拍广告，不夸张表情；不出现结果图、硬产品露出、文字、logo、水印。
```

### 3. 单步骤教学页

```text
用途：第 [n] 页，只解释一个能执行的动作。
画面：俯拍或近景，双手正在完成[单一步骤]；清楚看到[工具/材料/关键动作]；
上方留白放“步骤 n + 一句动作”，下方留小块说明时间、用量或注意点。
只出现本步骤所需物品，不展示后续步骤，不生成文字或编号。
```

### 4. 对比决策页

```text
用途：帮助读者按条件选择，不是硬广。
画面：A 与 B 两个[方案/产品/效果]在同一光线、同一视角、同一尺度下并列；
中间清晰分隔，顶部留白放“怎么选”，每一侧下方预留短标签区。
视觉客观、可核查、不过度美化；涉及前后对比时保持角度、光线、肤色/环境一致。
不要文字、夸张疗效、误导性反差、logo、水印。
```

### 5. 收束收藏页

```text
用途：最后一页，把内容压缩成可保存的行动条件。
画面：围绕[核心成果/真实物件]的极简场景，主体在下方或右下；
左上留 45% 纯净区域，供“3 个记住点 / 适用条件 / 下一步”短清单。
视觉安静、有完成感，与封面色盘一致，保留少量生活纹理；不要大段文字、二维码、强迫关注、促销海报感、logo、水印。
```

## 通过标准

- 封面和每页都有导演表；没有“画什么/字在哪/为什么”的页面不允许批量出图。
- 同一主题切换内容目标时，页面角色、构图和文字密度必须随之改变。
- 生图前后都检查：关键文字可读、主体不被挡、页面只讲一个重点、整组有节奏但不失一致性。
