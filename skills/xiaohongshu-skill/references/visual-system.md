# 小红书视觉系统

## 什么时候读

当任务涉及封面、内页、视觉路线、真实感、小图可读性或图片风格时读取本文件。

## 视觉路线由内容目标决定

先定页面任务和证据方式，再选路线。不是喜欢哪种风格就用哪种：

- 要共鸣或建立信任：真实场景、人物动作、过程痕迹。
- 要让人收藏或理解方法：步骤卡、对象细节、清晰层级。
- 要帮助比较决策：同视角对照、判断维度、适用边界。
- 要建立产品/服务理解：真实使用场景、产品细节、证据物。
- 要引发观点讨论：有张力的场景或一句大判断，但必须有内页兑现。

没有用户真实素材时，不是直接停图，而是按用户要的方向生成模拟真实素材。

## 真实感标准

真实感来自四层：

- 材料真实：用户提供或可公开使用的真实素材。
- 场景真实：桌面、门店、工具、流程、笔记、实物。
- 细节真实：光线、纸张、屏幕、手写痕迹、不完美边缘。
- 表达真实：承认限制，不夸张承诺。

用可被看见的生活和工作痕迹承担真实感。

## 手机小图标准

封面必须通过缩略图判断：

- 主标题最大，第一眼可读。
- 主体和文字不互相遮挡。
- 关键信息不放在右下角小贴纸。
- 画面不靠一堆小字解释。
- 颜色对比够，但不做廉价广告感。

## 内页一致性与页面节奏

整套图文通常是封面 + 6 到 8 页内页。内页应保持：

- 统一比例，默认 3:4。
- 统一标题层级。
- 统一色彩和组件。
- 每页只讲一个要点。
- 页面之间交替“场景 / 解释 / 证据”，不重复同一张图。
- 真实场景页少字，方法与比较页可多字；文字必须在留白、低细节区或信息底板，不能压住关键证据。
- 每页必须从 `page-art-direction-system.md` 取得页面角色、文字区、主体区和禁止遮挡区。

## 视觉决策记录

生成图片前记录：

- 已确认视觉路线。
- 封面字。
- 真实素材缺口。
- 模拟真实素材方向。
- Image2 是否可用。
- 用户是否允许批量出图。
- 每页视觉导演表：画什么、文字写什么、文字摆哪、为什么这样设计。

没有这些记录，只能给候选方案，不能直接生成最终图。
