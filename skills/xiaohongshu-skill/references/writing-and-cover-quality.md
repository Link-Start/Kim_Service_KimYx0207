# 写作与封面质量

## 什么时候读

当任务涉及去 AI 味、标题、封面字、正文、评论区、标签或发布前自检时读取本文件。

## 去 AI 味

正文要像真实人写给真实人看：

- 从具体场景开始，不从大概念开始。
- 保留犹豫、限制和真实边界，不装成万能结论。
- 少用“赋能、闭环、抓手、体系化、价值感、效率提升”等空话。
- 少用排比式鸡汤；只有所选叙事路线需要时才写具体转折，其他路线用方法、标准、证据或边界建立价值。
- 不夸张承诺，不写保证结果。

## 叙事路线按营销决策选择

不要把“场景 -> 转折 -> 动作 -> 收束”当成所有笔记的固定正文。它只是信任种草路线的一个选项。根据上游决策选择：

- 搜索/收藏型：问题识别 -> 判断标准 -> 方法或清单 -> 可保存结论。
- 痛点共鸣型：具体卡点 -> 被忽略的原因 -> 新理解 -> 低门槛动作。
- 比较决策型：常见误区 -> 对比维度 -> 适合/不适合谁 -> 选择建议。
- 信任种草型：真实场景 -> 过程证据 -> 克制结果 -> 产品或服务出现的位置。
- 观点讨论型：反常识判断 -> 真实例子 -> 反面边界 -> 可讨论的问题。

无论哪条路线，都要有可代入场景、可兑现内容和克制边界；不要写成课程广告或标准营销页。

## 标题不是套公式，而是匹配需求阶段

标题从“要抢的需求 + 读者阶段 + 可兑现证据”推导。至少在以下路线中比较，而不是只换同义词：场景痛点、搜索答案、判断标准、反差观点、低风险行动。没有上游决策来源的标题，不得作为首选。

## 标题要求

完整笔记至少给 8 个标题备选。标题要满足：

- 读者一眼知道和自己有什么关系。
- 不虚假制造焦虑。
- 不承诺收入、结果、平台推荐或立刻变强。
- 不只堆热词。
- 与正文和图片页兑现一致。

## 封面字要求

完整笔记至少给 3 组封面字。封面字要满足：

- 主标题 6 到 12 字优先。
- 手机信息流缩略图先读主标题。
- 不依赖角标、小贴纸、小字补丁。
- 不把“真实学习桌面”“过程记录”之类说明文字当主标题。
- 封面承诺必须在正文和内页兑现。

## 标题封面兑现检查

检查三问：

- 标题承诺是什么？
- 封面第一眼承诺是什么？
- 正文和图文页有没有兑现？

如果标题说“避坑”，正文必须有具体坑和处理方式。如果封面承诺一个结果，内页必须展示目标、输入、步骤或验收。

## 封面、标题、正文与图片的分工

- 封面：让对的人一秒停住，只给一个冲突或承诺。
- 标题：补足“为什么与你有关”以及必要的搜索语义。
- 第 1 页：让读者确认“这说的是我”或先看到结果。
- 中间页：解释、证明、比较或给方法；每页只完成一个认知动作。
- 最后一页：压缩行动条件、适用边界或可讨论的问题。
- 产品/服务：由上游决定隐性出现、在过程里出现，还是明确出现；不能为了转化硬塞进每页。

## 互动边界

公开评论区可以回复方法、边界和公开可核验信息。不要主动引导私信、报名、加群、领取资料、外链或联系方式。用户主动询问课程/服务时，也只回复公开信息、适用人群和限制，不承诺效果。
