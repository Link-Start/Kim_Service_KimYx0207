# 领域与材料判断

## 什么时候读

当用户的需求很模糊、材料不足、主题涉及课程/服务/产品/咨询，或需要判断能不能写成小红书笔记时读取本文件。

## 先判断四件事

1. 这条笔记给谁看。
2. 读者现在卡在哪里。
3. 用户手里有什么真实材料。
4. 哪些内容不能编。

没有真实材料时，不要直接放弃图片。用户要模拟真实素材，就继续进入 Image2 图片 Brief，用生成图承担真实感、场景感和图文表达。

## 内容机会研究（由 Skill 完成）

不要问用户“流量从哪里来”。用户不需要懂搜索词、内容竞争或互动数据；Skill 要根据主题、产品、目标人群和已有材料自行完成：

1. 生成读者会使用的搜索词、场景词、痛点词、比较词和行动词。
2. 用当前可用的检索能力看官方平台资料、公开可见的同类内容和讨论，提取重复承诺、读者常问问题、可验证证据与未被满足的角度。
3. 判断读者在未意识到问题、已知痛点、找方法、比较选择、准备行动中的哪一阶段。
4. 明确这条内容要抢的需求，以及不能碰的过度重复套路或不能证实承诺。

每个结论标注 `用户提供`、`研究证据` 或 `AI 假设`。没有检索能力或无法验证互动量时，写研究缺口；不得编造“高赞”“爆款”或假数据。

## 常见任务类型

- 课程/训练营：强调适合谁、怎么学、边界是什么，不承诺速成。
- 咨询/服务：强调问题识别、过程、适用边界，不硬导私信。
- 产品/工具：强调使用场景、真实限制、可验证结果，不夸大效果。
- 个人 IP：强调具体经验和克制表达，不造人设。
- 本地商家：强调真实场景、服务细节和消费边界，不编客户评价。

## 材料分级

- 已提供：用户给的原文、截图、图片、课程大纲、产品说明、案例素材。
- 可合理生成：桌面、信息卡、流程图、抽象示意、脱敏学习场景、通用校园/留学生活场景。

## 从机会到创意决策

领域判断和内容机会研究必须形成一条推导链：

```text
目标读者 -> 要抢的需求 -> 内容目标 -> 证据强度 -> 表达强度
-> 标题承诺 -> 正文叙事 -> 封面任务 -> 逐页画面与文字位置
```

使用同目录的 `marketing-decision-model.md` 和包内 `../assets/marketing-decision-board.md` 记录此链。研究只能改变后续创意决策；不能改变时就停止堆资料。
