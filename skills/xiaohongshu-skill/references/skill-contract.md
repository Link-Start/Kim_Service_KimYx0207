# 小红书图文技能合同

本文件是技能的权威契约来源。决策状态、图片路线硬规则、降级边界、前置决策顺序都以本文件为准；其他 reference 出现同样概念时引用本文件，不重复定义。

## 结果定义

本技能产出的是可发布前验收的完整小红书/Rednote 图文包：机会研究、营销决策、标题、封面字、正文、评论边界、标签、封面和 6-8 页内页、逐页视觉导演、图片路线、图片证据或阻塞说明都要清楚。

## 触发合同

只有用户明确需要完整小红书/Rednote 图文笔记、包含封面和内页的发布包、或整套图文视觉制作时触发。

不触发：

- 只改一句标题。
- 只要标题组、封面字、正文、评论回复、标签或单个 Image2 提示词。
- 只做普通长文润色。
- 只做 PPT、邮件、论文或代码。

## 输入合同

优先读取：

- 用户原始输入或 `codexDelegation.input`。它高于 HookPrompt / 优化提示词 / 模型总结。
- 主题和受众。
- 用户真实材料。
- 产品/课程/服务边界。
- 是否有图片素材。
- 是否允许生成图片。
- 不能写的事实。

搜索需求、内容竞争、公开讨论和高信号样本由 Skill 自行研究；用户不需要也通常不会知道“流量从哪里来”。检索不可用时必须标记研究缺口，不得编造热门案例或互动数据。

材料缺失时必须列出缺口，不用想象补齐。

如果优化提示词或模型自写 scope 中出现“只输出文案”“不需要制作图片”“不生成图片”，但用户原始输入没有明确这么说，视为提示词污染，不得作为执行边界。

## 输出合同

完整工作台按 15 段结构输出并可写入归档文件，15 段定义见 `evaluation-method.md`。在策略段和图文页脚本中，必须额外记录机会研究、营销决策和逐页视觉导演；完整包不允许以局部交付替代。

最终聊天里要能看到可复制的标题、正文、互动边界、标签和本轮新生成图片结果；文件只是补充证据。最终聊天默认使用发布版交付结构（定义见 `SKILL.md`），不把完整 15 段工作台整段贴出，除非用户明确要求。

## 决策状态合同（权威定义）

整个技能只用三种决策状态，其他文件引用本节，不再重新定义：

1. `human_confirmed`：用户原话选择，或 Codex `request_user_input` / Claude Code `AskUserQuestion` / 宿主等价原生选择工具返回。模型推荐不等于本状态。
2. `explicit_auto_permission`：用户明确说“你定/直接做/不用问/你看着来并执行”，或原始输入已经是“帮我生成 / 做一组 / 适合发布 / 图文 / 照片或画面加文字”等生成动作且没有要求先停、只要文案或等确认。输出必须记录 AI 选择理由和风险。本状态最终必须列出可替换项和安全声明。
3. `blocked_pending_human_decision`：缺关键人类决策，且没有原生确认工具或未返回，必须停止，不得生成图片或完整交付。

`默认假设` 不是继续执行许可。默认假设只能用于展示方案、标题备选、视觉备选和待确认项，不能触发 Image2 生成、批量出图或交付级声明。

## 图片路线合同（权威硬规则）

路线优先级从高到低，不可跳级：

1. Image2 / 宿主原生图片生成。
2. 其他图片生成 MCP 或插件（豆包、即梦、MiniMax、Midjourney 等）。
3. 已有真实素材处理。
4. SVG、HTML、本地排版 PNG、静态结构预览。

硬约束：

- **当第 1 级或第 2 级任一可用时，禁止使用第 4 级（SVG / HTML / 本地排版 / 静态结构图）**。只有第 1 级和第 2 级都不可用、都失败，或用户明确要求可编辑文字时，才允许进入第 4 级，并必须写明 fallback 原因和降级证据。
- “文字信息卡”“图片上文字说明是主体”“大字封面”“知识卡”**不是**跳过 Image2 的理由，也不是直通本地排版的理由。这类请求仍先用 Image2 / 宿主原生做一体化图文封面；只有中文错字、不可读、排版失败或用户要求可编辑时，才降级本地排版。
- 调用其他图片 MCP（第 2 级）前，必须先写 `Image2 优先级证据`：Image2 已调用并失败 / 当前宿主无 Image2 可调用工具 / 用户明确要求不用 Image2。没有证据就不得调用 MCP，也不得调用本地排版。
- SVG、HTML、静态图只能是结构预览，不能冒充真实图片，不能写成交付级。
- 旧 `outputs/`、旧线程图片、示例文件和缓存图不能证明本轮已生成图片，除非用户明确要求复用。

## 人类决策合同

以下节点需要用户确认或明确授权（决策状态见上文「决策状态合同」）：

- 策略路线。
- 标题和封面字。
- 视觉路线。
- 图片 Brief。
- 封面候选。
- 是否批量生成内页图。

没有确认时，只能给候选方案，不能直接生成最终图文包。用户已经要求图文/生成/引流贴且前置方向确认完成后，不要再把“只做文案/不生成图片”作为推荐确认项；继续生成封面候选，封面通过后再确认批量内页。

用户通过原生弹框确认批量内页后，生成路线已经锁定为完整图片包。此时不得再问“做到哪一步”、不得输出“文案+脚本(推荐)”、不得停在单张封面；除非图片工具失败，只能继续生成全部计划页并在最终聊天列出本轮新图片证据。

## 可选视觉工作台合同

视觉工作台是图片路线之外的可选承载层。只有机会研究、营销决策、逐页视觉导演和对应 Image2 Brief 已完成，且首张本轮 3:4 封面候选已经可见后，才能询问是否启用；它不得改变上文图片路线优先级，也不得被算作第 2 级图片生成器。

- 封面阶段只承载 3:4 槽位和本轮新封面资产，用于手机缩略图检查；通过后仍必须由宿主原生选择工具取得批量许可。
- 批注返修必须保留原图，把修订图放在原图旁边，重新做缩略图和内容检查，并记录明确的 `accepted_asset`。画布页、槽位或 shape id 只能证明放置，不能证明图片已生成、质量已通过或用户已接受。
- 批量完成后必须形成逐页资产清单；只返修失败页，不得因为单页失败重新生成整组。
- 工作台不可用时，Codex/Claude Code 继续使用原有原生决策工具与图片路线，不得因此阻塞，也不得改变触发范围。
- 同一价值阶段最多询问一次；只有出现新的 before/after 封面候选或明确失败页返修候选时，才能凭新 artifact hash 和新价值证据跨阶段再问。无原生选择面时 Markdown 卡只作降级等待，不能冒充弹窗。
- 状态区分 `not_offered`、`offered_pending`、`accepted`、`declined`、`unavailable`、`launch_failed`、`opened_verified`。原生选择返回证明确认；画布状态回读证明实际落盘，两者不能互相替代。

字段、阶段和证据边界见 `visual-workspace-contract.md`；供应商能力只在独立适配清单中声明，不写进本合同的控制规则。

## 平台边界合同

- 不主动引导私信、加群、报名、领取资料、外链或联系方式。
- 不承诺学习、成交、涨粉、变现或效果。
- 课程、咨询、服务内容要写清适用人群和限制。

## 验收合同

验收时检查：

- 标题和封面是否兑现。
- 正文是否有人味，并兑现上游选定的信任、搜索收藏、比较咨询或观点讨论路线；转折只在适合的路线中检查。
- 图片路线是否有证据。
- 缺实图时是否诚实标注。
- 聊天最终答复是否是发布版，而不是 15 段工作台倾倒。
- 图片证据是否来自本轮生成，而不是旧包复用。
- 评论和私信边界是否合规。
- 15 段结构是否完整（定义见 `evaluation-method.md`）。
- 是否有从机会研究到标题、封面和逐页导演表的可追溯推导。
