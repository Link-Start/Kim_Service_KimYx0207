## 分层与模块选择

这不是默认目录树。先按 `core / conditional / release` 选择模块；未命中条件的模块不创建，也不保留空目录。

| 层级/模块 | 选择触发条件 | 本次选择 | 具体消费者 | 防止的失败 | 产物或验证 |
|---|---|---|---|---|---|
| `core: SKILL.md` | 所有可复用 Skill | 必选 | 宿主路由与执行者 | 无触发、无步骤或无边界 | 轻入口 |
| `conditional: references` | 详细规则无法在入口内清楚表达 |  | 具体执行步骤 | 领域规则被猜测 |  |
| `conditional: assets` | 多次运行需要复用材料或工作表 |  | 具体产物步骤 | 每次重造、格式漂移 |  |
| `conditional: scripts` | 存在可确定校验、转换或构建 |  | 执行/验证步骤 | 纯叙述无法复现 |  |
| `conditional: evals` | 触发、边界或输出需要回归 |  | 验证步骤 | 改动后行为漂移 |  |
| `conditional: examples` | 一个小实例能显著降低误用 |  | 初次使用者 | 抽象规则难落地 |  |
| `release: public files/evidence` | 准备公开、迁移或销售 |  | 外部用户与发布门禁 | 无许可、无变更记录、声明大于证据 |  |
