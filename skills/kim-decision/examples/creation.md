# Creation Example

## Visible output

Build the artifact around one remembered line, not around a complete taxonomy. If the receiver cannot repeat the point after the first screen, more sections will only make the failure heavier.

The first version should feel like this: one sharp claim, one concrete example, one action the receiver can take immediately. The receiver should think, "I know what this is for, and I know what to do next."

## Plan

Create a 200-word version before the full artifact.

- Input: one claim, one example, one intended receiver.
- Action: write the shortest version that moves the receiver from seeing to acting.
- Output: a draft with a headline, a proof/example, and one next action.
- Pass condition: the receiver can repeat the claim and name the next action.
- Kill condition: if the receiver calls it "interesting" but cannot say what it changes, the core promise is weak.

## Author check only, not user output

- Essence: one controlling idea.
- Narrative: start from consequence or conflict.
- Evidence: one concrete example beats broad explanation.
