# 校准示例

## 可见输出

第一个问题不是文案不够好，而是接收者读完不知道下一步做什么。先只修这个断点。

把那一段改成：一个动作、一个输入、一个成功信号。比如：“把当前草稿发给一个真实接收者，让他标出第一句让他停住的地方。如果两个人都停在同一句，先改那一句，别动其他部分。”

## 直接用

今天只改第一个断点，不重做整个系统。

通过标准：一个接收者不追问，也能说出下一步动作。

停止标准：如果三个人停在三个不同位置，问题就不是某句话，而是整个承诺不清楚。

## 后台检查

- 路径：接收 -> 理解 -> 行动 -> 得到信号。
- 杠杆点：先降低选择负担，再谈风格优化。
- 证据缺口：断点数量未知，所以先测一段，不做全量重写。
