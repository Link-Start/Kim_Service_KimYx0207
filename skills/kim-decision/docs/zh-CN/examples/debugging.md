# 排错示例

## 可见输出

先别再改方案。重复失败说明现在缺的不是努力，而是证据。

先抓最小复现：什么输入进来、出现什么输出、最近改了什么、原本哪个假设应该成立。真正有用的时刻，是失败从一种感觉变成一条记录。

## 方案

改动前先跑一次证据检查。

- 输入：失败案例、上一次正常案例、最近变化。
- 动作：对比两者，写一个可证伪假设。
- 输出：复现记录，加一个定向修复或回滚动作。
- 通过标准：同一个输入能连续两次产生同一个失败。
- 停止标准：如果复现不了，先补日志或观察点，不继续修。

## 后台检查

- 根因门：现象、复现、最近变化、证据、假设、最小验证。
- 风险：大范围重写会把原因藏起来。
- 反馈：每一次修复都必须比上一次产生更清楚的信号。
