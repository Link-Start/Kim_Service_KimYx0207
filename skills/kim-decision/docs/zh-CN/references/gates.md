# 硬门

## 路径硬门

没有路径，不写完整方案。

必须有：

```text
主体
动机
理解
行动
阻力
信号
状态变化
延续动作
```

## 证据硬门

依赖下面内容时，必须查证或标未确认：

```text
时间
外部规则
外部系统
私有文件
高风险判断
当前市场状态
```

## 最小验证硬门

没有最小验证，不写大系统。

必须有：

```text
目标
输入
动作
输出
通过标准
失败信号
下一步
不做什么
```

## 无占位符硬门

不要留占位内容。

避免：

```text
待定
稍后
适当处理
视情况优化
提升体验
完善闭环
```

写清：

```text
执行者
输入
动作
输出
通过标准
```

## 根因硬门

重复失败时，先收集证据，再改方案。

必须有：

```text
现象
复现路径
最近变化
证据
假设
最小验证
```

## 完成硬门

说完成之前检查：

```text
验收标准
未确认事实
未处理风险
占位内容
可用结果
```

## 三次失败硬门

同一问题连续失败三次，停止继续改。

回到下面内容重新判断：

```text
意图
路径
根因
模型选择
缺失输入
范围
替代路径
```

## 研究硬门

当关键决策依赖 C 或 D 级证据时，必须先尝试验证。

步骤：

1. 找出哪些 C/D 证据如果升级会改变决策
2. 用可用工具搜索：联网搜索、文件读取、API查询、文档查找
3. 找到 → 升级等级（C→B 或 D→C），标注来源
4. 找不到 → 标记"未验证"，写明"无法验证，需用户确认"

规则：

- 关键决策不能只靠 D 级证据
- 每条关键 C/D 证据至少尝试验证一次
- 没有搜索工具时，跳过搜索但明确标记缺口

## 挣钱硬门

加载：references/business.md

必须有：

```text
挣钱模式
付款人
为什么现在付
成交金额
交付成本
能不能复购
```

问题 1-3 答不出 → 降级为"不建议推进"。

## MVP硬门

加载：references/business.md

必须有：

```text
V1范围（一句话）
明确不做什么
最快交付天数
最小客户
最小成交额
失败标准
```

范围超过一句话或没有排除项 → 压缩。

## 交付硬门

加载：references/business.md

处理
输出
