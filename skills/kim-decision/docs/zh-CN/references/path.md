# 路径

## 结构

```text
主体 -> 动机 -> 理解 -> 行动 -> 阻力 -> 信号 -> 状态变化 -> 延续动作
```

## 主体

谁经历这个结果？

好的主体：

```text
必须做决定的人
接收草稿的人
运行流程的人
进入产品的人
```

## 动机

主体为什么现在进入？

常见来源：

- 痛点
- 欲望
- 压力
- 好奇
- 义务
- 风险
- 收益预期

## 理解

主体第一眼理解什么？

检查：

- 是否知道这是什么
- 是否知道为什么和自己有关
- 是否知道下一步做什么

## 行动

第一个有意义动作是什么？

它必须足够小、可见、可测试。

## 阻力

什么阻止行动？

常见阻力：

- 承诺不清
- 选择太多
- 信任不足
- 操作成本高
- 缺少证据
- 害怕损失
- 反馈太慢

## 信号

行动后出现什么反馈？

信号可以是回复、点击、付款、报错、测试结果、使用事件、保存的草稿、状态变化。

## 状态变化

信号出现后发生什么变化？

例子：

- 未知变已知
- 怀疑变信任
- 模糊想法变决策
- 卡住流程变可运行流程

## 延续动作

下一步是什么？

例子：

- 放大
- 停止
- 修订
- 购买
- 分享
- 复用
- 验证
- 交接

```markdown
路径断点：

```
