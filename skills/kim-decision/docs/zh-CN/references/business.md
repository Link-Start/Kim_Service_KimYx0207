# 商业层

当任务涉及方案、商业决策、变现、产品策略、定价、客户交付，或任何有商业维度的决策时加载。

触发关键词：方案、商业、变现、定价、客户、交付、产品决策、挣钱、复购、老板、MVP、收入、成本、revenue、monetize、price、client、deliver、scope。

## 挣钱判断器（硬门）

任何商业输出完成前，必须回答全部六个问题：

1. 挣钱模式：怎么挣钱？
2. 付款人：谁付钱？
3. 紧迫性：为什么现在付？
4. 成交金额：多少钱能成交？
5. 交付成本：履约成本是多少？
6. 复购：能不能复购？

降级规则：

- 问题 1-3 答不出 → 输出降级为"不建议推进"，列出具体缺口
- 问题 4-6 答不出 → 标记"需验证"，列出具体缺口
- 六个都答了 → 输出收入估算，继续

## MVP收敛器（硬门）

每个方案强制压缩范围：

1. V1 范围：第一版只做什么？（最多一句话）
2. 排除项：明确不做什么？（至少三项）
3. 最快交付：几天能上线？
4. 最小客户：谁是最小可用客户？
5. 最小成交：多小的交易额能验证模式？
6. 失败标准：什么结果意味着"停，这条路不通"？

压缩规则：

- V1 范围超过一句话 → 压缩到一句话
- 没有排除项 → 范围太宽，砍到有排除项为止
- 最快交付超过 14 天 → 拆成更小的块
- 没有失败标准 → 补上再继续

## 老板视角（模型）

从买单人角度审视方案：

1. 购买理由：解决了预算持有人的什么痛点？
2. 员工采纳：员工会不会主动用，不用逼？
3. 实施难度：从"买"到"第一次出结果"要几步？
4. 快速见效：第一周有什么可见效果？
5. 不买的损失：不买会怎样？

输出：

```text
发现：[老板会质疑或拒绝的点]
修正：[怎么解决]
```

规则：如果没有快速见效项，方案只靠长期承诺。标记"需要短期可见结果"。

## 交付闭环（硬门）

每个交付物必须闭环：

1. 输入：什么进来？（格式、来源、质量标准）
2. 处理：发生什么？（步骤、工具、时间）
3. 输出：什么出来？（格式、去向、质量标准）
4. 验收标准：客户怎么验证完成了？
5. 客户确认：客户签字、发邮件、说什么话来确认？
6. 复用：什么能被下一个客户/项目复用？

规则：

- 1-5 缺任何一项 → 交付物未就绪
- 6 缺失 → 标记"一次性工作，无杠杆"
- 复用可以是：模板、流程、工具、脚本、检查清单、框架

## 与核心框架的关系

- 挣钱判断器：在证据之后、输出之前运行
- MVP收敛器：在路径之后、最小验证之前运行
- 老板视角：作为模型与其他模型一起运行
- 交付闭环：作为最后的硬门，在输出之前运行
